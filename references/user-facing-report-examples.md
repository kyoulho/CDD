# User-Facing Report Examples

이 문서는 CDD 사용자 보고 예시가 더 필요할 때만 읽는다. 기본 응답 규칙은 `_user-facing-language.md`에 있고, 이 파일은 예시 reference다.

## 1. 아직 진행하면 안 되는 경우

Bad:

```text
Product Readiness: NOT READY
Engineering Readiness: NOT READY
DB_DESIGN_BLOCKED
API_DESIGN_BLOCKED
STATE_MODEL_BLOCKED
```

Good:

```text
확인한 기준:
- 현재 확인한 기준 문서와 요청 내용

현재 판단:
- 아직 결정 필요

이유:
- 첫 기능 범위가 아직 정해지지 않았습니다.
- 사용자가 어떤 행동을 하고 어떤 결과를 받는지 정해지지 않았습니다.
- 무엇을 구조화해서 저장하고 무엇을 자유롭게 적게 할지 정해지지 않았습니다.
- 상태값이 무엇을 의미하는지 정해지지 않았습니다.
- 데이터가 많아졌을 때 조회와 권한 처리를 어떻게 할지 정해지지 않았습니다.

먼저 정할 것:
1. 첫 버전의 기능 범위
2. 사용자가 시작하는 위치와 입력 방식
3. 성공, 실패, 빈 상태에서 보여줄 결과
4. 저장할 정보와 저장하지 않을 정보
5. 상태값을 둘지, 둔다면 각 상태의 의미

내 추천:
- 먼저 기능 범위와 저장 이유를 정한 뒤 저장 방식과 동작 방식을 설계합니다.

다음에 할 일:
- 위 결정이 끝나면 작업 기준서를 만들 수 있습니다.

내가 물어볼 것:
1. 첫 버전에서 반드시 포함할 행동은 무엇인가요?
2. 사용자가 나중에 다시 확인해야 하는 정보는 무엇인가요?
```

## 2. 사용자 개입 없이 진행 가능한 경우

Bad:

```text
Prompt Draft Approval is not Prompt Execution Approval.
```

Good:

```text
사용자 선택이 필요한 부분은 없습니다.
작업 지시서는 내부 실행 기준으로 작성하고, 새로 결정할 사항이 없으므로 같은 요청 범위 안에서 바로 실행합니다.

진행할 작업:
- 구현 지시서 작성
- 승인된 기준 안의 구현
- 검증 결과 보고

진행하지 않을 작업:
- 기준 문서 변경
- 데이터 삭제
- public API 제거
```

## 3. 완료한 경우

Good:

```text
이번 작업은 완료되었습니다.

확인한 결과:
- 요청한 문서 변경이 반영되었습니다.
- 검증 명령이 통과했습니다.
- 남은 미확정 결정은 없습니다.

다음 후보:
1. 다음 작업 기준서 작성
2. 구현 지시서 작성
3. 검증 기록 정리

내 추천:
- 바로 이어질 작업이 정해져 있다면 구현 지시서를 작성합니다.
```

## 4. 요청이 애매한 경우

Bad:

```text
Work Mode가 불명확합니다.
ANALYSIS_ONLY인지 IMPLEMENTATION인지 확인해 주세요.
```

Good:

```text
지금은 실제 수정을 시작하기 전에, 먼저 어느 단계까지 원하는지 확인해야 합니다.
어느 쪽으로 진행할까요?
1. 현재 상태를 읽고 판단만 한다
2. 문서 초안을 만든다
3. 문서를 실제로 수정한다
4. 구현 계획만 만든다
5. 코드를 실제로 수정한다
6. 삭제/정리 대상을 먼저 목록화한다
```

## 5. 모호한 승인

Bad:

```text
"좋아"라고 하셨으니 구현을 시작하겠습니다.
```

Good:

```text
아직 실제 구현을 시작해도 된다는 승인은 명확하지 않습니다.
먼저 구현 계획만 볼지, 실제 코드 수정을 원하는지 확인해 주세요.
```

## 6. 선행 작업이 끝나지 않은 경우

Bad:

```text
TASK-002 prompt legitimacy check failed due to unresolved predecessor gate.
```

Good:

```text
TASK-002 지시서 파일은 있지만, 선행 작업인 TASK-001이 아직 끝나지 않아 지금 기준으로는 사용할 수 없습니다.
먼저 TASK-001의 남은 결정을 정리해야 합니다.
```

## 7. 기준 문서 변경이 필요한 경우

Bad:

```text
BLOCKED_BY_POLICY_CONFLICT. Source of Truth Change Request required.
```

Good:

```text
지금 요청은 기존 기준 문서와 충돌합니다.
구현으로 바로 처리할 수 없고, 먼저 기준 문서를 바꿀지 결정해야 합니다.
변경하려면 아래를 먼저 정해 주세요.
1. API 응답만 바꿀지
2. 내부 도메인 용어도 바꿀지
3. 관련 작업 지시서까지 함께 바꿀지
```
